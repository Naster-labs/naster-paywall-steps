# Security Policy

We care about the security of users and contributors.

## Supported Versions

This is an educational PoC and does not provide security patches or SLAs.  
We accept reports and patches on a best-effort basis.

## Reporting a Vulnerability

- Email: **lukasz.luczak@luczak.consulting**
- Alternatively, open a **private** security advisory on GitHub (Security → Advisories).

Please include:

- Affected version/commit, environment, and reproduction steps
- Impact assessment (worst case)
- Suggested mitigation if you have one

We aim to acknowledge reports within **5 business days**.

## Disclosure

- We prefer **coordinated disclosure**.
- We’ll credit reporters if desired.
- Do **not** test against production systems you don’t own.

## Scope Notes

This repository is a **learning project**. Do **not** use it as-is in production.
