# Contributing

Thanks for your interest!

## Development

- Use Node 22 (devcontainer provided) and Yarn Berry
- Each folder in `steps/` is a complete runnable snapshot
- Keep steps atomic and well-documented

## Pull Requests

- Follow **Conventional Commits** (`feat:`, `fix:`, `docs:`, `chore:`, …)
- Add/update README for a step when you change it
- Run `yarn lint` and `yarn check-format` before pushing

## Code Style

- ESLint (flat config) + Prettier
- SPDX header `SPDX-License-Identifier: MIT` at the top of source/docs

## Security

- See `SECURITY.md` — report privately, no public 0-days please.
